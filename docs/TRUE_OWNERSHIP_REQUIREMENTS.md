# True Ownership Requirements

True ownership means Jarvis is the phone, not an app on the phone.

| Requirement | Classification |
| --- | --- |
| Starts automatically | possible through supervision |
| Cannot be casually exited | possible through supervision |
| Controls home experience | possible through supervision |
| Controls lock experience | possible through jailbreak only |
| Handles camera, mic, sensors, storage, memory | possible through native app |
| Has voice identity | possible through native app |
| Has local tools | possible now |
| Has offline brain | possible now |
| Can open or replace core flows | possible through supervision |
| Can survive reboot or relaunch | possible through supervision |
| Can recover safely | possible through supervision |
| Has uninstall and recovery path | possible through supervision |
| Has permission and safety policy | possible now |
| Optionally controls other apps | possible through jailbreak only |
| SpringBoard hooks | possible through jailbreak only |
| Lock screen hooks | possible through jailbreak only |
| Global button remaps | possible through jailbreak only |
| Firmware replacement | impossible or not practical |

## Meaning

Native code can handle Jarvis tools. Supervision can make the device behave like an appliance. Jailbreak is required for true system ownership. Firmware replacement is not a practical path for iPhone XR.
