# Control Mesh Limits

What works without jailbreak:

- JARVIS can run its own native UI.
- JARVIS can handle local commands.
- JARVIS can speak through local system speech.
- JARVIS can open its own camera inspection screen.
- JARVIS can receive `jarvis://` deep links.
- Shortcuts can send commands into JARVIS.
- Voice Control can operate visible iOS UI.
- Vocal Shortcuts can trigger configured actions.
- Guided Access can keep JARVIS foreground after setup.

What remains jailbreak-only:

- SpringBoard hooks.
- Lock screen hooks.
- Root daemons.
- Hidden arbitrary screen reading.
- Injected taps into other apps.
- Global button remaps.
- True OS takeover.

Best current path:

Use JARVIS plus Voice Control, Vocal Shortcuts, Shortcuts, URL scheme links, optional App Intents, and Guided Access.
